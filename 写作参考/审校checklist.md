# 三遍审校 Checklist

> 每次写完初稿，按顺序完成三遍审校。
> AI 在执行审校时，会逐一勾选并说明修改内容。

---

## 第一遍：内容审校（逻辑 · 事实 · 结构）

**目标**：确保文章内容准确、逻辑清晰、无编造

- [ ] 所有数字和数据是否真实？（来源可查）
- [ ] 所有时间节点是否准确？
- [ ] 所有产品名称、人名是否正确？
- [ ] 有没有编造的案例或经历？
- [ ] 前后文有没有逻辑矛盾？
- [ ] 有没有偏离主题的段落？
- [ ] 文章结构是否清晰（开头-中间-结尾）？

**完成标准**：以上全部勾选，或标注"已核实/不适用"

---

## 第二遍：风格审校（去 AI 味 · 增加人味）

**目标**：让文章读起来像人写的，而不是 AI 写的

### 🗑️ 删除套话
- [ ] 搜索并删除：`在当今`、`随着`、`综上所述`、`值得注意`、`不可忽视`
- [ ] 搜索并删除：`毋庸置疑`、`显而易见`、`不言而喻`
- [ ] 搜索并删除：`对此`、`为此`、`有鉴于此`（书面过渡词）

### 🔧 拆解 AI 句式
- [ ] 检查 `"不是…而是…"` 是否连续出现超过2次 → 拆散
- [ ] 检查是否有超过30字的长句 → 拆成短句
- [ ] 检查是否有大量 `"首先…其次…最后…"` 结构 → 改成更自然的过渡

### 📝 替换书面词汇
| 原词 | 替换方向 |
|------|---------|
| 显著提升 | 具体数字（"快了3倍"）|
| 充分利用 | "用好"、"用上" |
| 有效实现 | 直接说做到了什么 |
| 进行操作 | 直接用动词 |
| 开展工作 | 直接说做什么 |
| 具有…特性 | 直接说它能做什么 |

### 💬 增加人味
- [ ] 有没有明确的个人观点和态度？（不是中立客观）
- [ ] 有没有具体的真实细节？（数字、场景、感受）
- [ ] 有没有至少1处真实的个人经历或案例？
- [ ] 结尾是否表达了明确立场，而不是模糊总结？

**改写示例参考**：
```
❌ "在当今AI技术飞速发展的时代，工具也在不断进化..."
✅ "这个工具出了。我用了两周，确实好用。"

❌ "通过充分利用该工具，可以显著提升工作效率..."
✅ "用好这个工具，你每个项目能省2-3小时。我测了，大概是这个数。"
```

---

## 第三遍：细节打磨（标点 · 排版 · 节奏）

**目标**：让文章读起来顺畅，在手机上阅读体验好

### 句子长度
- [ ] 扫描全文，找出超过30字的句子 → 拆成两句
- [ ] 主体陈述句控制在15-25字
- [ ] 有没有长句短句交替，制造节奏感？

### 段落长度
- [ ] 每段不超过5行（手机屏幕）
- [ ] 一个意思一个段落，不要把两个意思塞进一段

### 标点使用
- [ ] 多用句号，少用逗号连接长句
- [ ] 感叹号不超过3个（用多了失效）
- [ ] 引号使用是否准确？

### 节奏感
- [ ] 大声朗读一遍 → 卡顿的地方就是需要修改的地方
- [ ] 有没有连续5句以上长度相似的句子？（变换节奏）
- [ ] 开头3句是否有力？
- [ ] 结尾是否有收束感，而不是草草结束？

---

## 审校完成后

- 初稿：`_drafts/项目名-初稿-v1.md`
- 审校后：`_drafts/项目名-初稿-v2.md`（或更新v1）
- 最终稿：`_published/项目名-final.md`
