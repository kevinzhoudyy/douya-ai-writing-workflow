# CLAUDE.md — Prompt 梳理工作流

> 进入此目录前，已完成根目录 CLAUDE.md 的工作区路由判断。
> 本文件定义 Prompt 整理、优化和管理的工作流。

---

## Prompt 管理任务类型

| 类型 | 描述 | 处理方式 |
|------|------|---------|
| **整理** | 把零散的 Prompt 归类、命名、存档 | 分析→分类→存档 |
| **优化** | 改进已有 Prompt 的效果 | 分析问题→重写→说明改动点 |
| **新建** | 根据需求从零写一个 Prompt | 明确目标→设计→测试建议 |
| **复盘** | 分析某个 Prompt 为什么效果不好 | 诊断→分析→改进方案 |

---

## Prompt 整理流程

### Step 1：理解待整理内容

用户提供散乱的 Prompt 集合后：
1. 快速扫描，理解大概有哪些类型
2. 告知用户：发现了X个Prompt，大概分为X类
3. 请用户确认分类方向

---

### Step 2：分类方案讨论

**AI提供分类建议**（按实际内容）：

常见分类维度：
- 按用途：写作类 / 编程类 / 分析类 / 翻译类 / 其他
- 按质量：已验证有效 / 待验证 / 需要优化
- 按平台：Claude / GPT / Midjourney / 其他

**等待用户确认分类方式后再整理。**

---

### Step 3：整理归档

每个 Prompt 的标准档案格式：

```markdown
# Prompt 名称

## 基本信息
- 用途：
- 适用模型：
- 最后验证日期：YYYY-MM-DD
- 效果评级：⭐⭐⭐⭐（1-5星）

## Prompt 正文
```
[Prompt 内容]
```

## 使用说明
- 使用场景：
- 注意事项：
- 示例输出：

## 版本记录
- v1.0 (YYYY-MM-DD)：初始版本
```

保存路径：`_prompts/分类名/Prompt名称.md`

## Prompt 优化流程

### Step 1：诊断现有 Prompt

分析原 Prompt 存在的问题：
- 目标不清晰（AI不知道要做什么）
- 缺少上下文（AI猜测用户意图）
- 格式不明确（输出结果混乱）
- 约束不够（AI"自由发挥"过多）
- 过于复杂（一个Prompt干太多事）

### Step 2：重写方案

提供2个改进版本（不同策略），并说明每个版本的改动点和预期效果改善。

### Step 3：测试建议

提供3-5个测试用例，帮助用户验证优化效果。

## Prompt 设计原则

| 原则 | 说明 |
|------|------|
| **角色设定** | 给AI一个明确的身份（"你是一个...专家"） |
| **任务明确** | 一个Prompt只做一件事 |
| **格式指定** | 明确要求输出格式（列表、表格、Markdown等） |
| **约束清晰** | 说清楚不要做什么 |
| **示例驱动** | 复杂任务提供输入输出示例（Few-shot）|
