# CLAUDE.md — 小红书写作流程

> 进入此目录前，已完成根目录 CLAUDE.md 的工作区路由判断。
> 本文件定义小红书图文笔记与纯文字帖的写作流程。

---

## 小红书写作的核心特点

| 特点 | 说明 |
|------|------|
| **封面标题最关键** | 封面标题决定是否有人点进来，比正文更重要 |
| **前三行定生死** | 展开前只显示3行，必须在前三行抓住读者 |
| **表达要有平台感** | 可用 emoji、短句、列表、提醒句增强阅读节奏，但是否使用取决于内容本身 |
| **分段要极短** | 每段1-2句话，大量换行，便于手机阅读 |
| **真实感最重要** | 读者极度敏感于"广告感"和"AI感"，真实经历最有力 |

### Step 1：理解需求 & 保存 Brief

**明确内容**：
1. 笔记类型（种草 / 测评 / 教程 / 经验分享 / 好物推荐）
2. 输出模式（图文笔记 / 纯文字帖）
3. 是否为商单（商单需要注意植入比例和软硬程度）
4. 目标受众（她/他是谁，有什么痛点）
5. 核心卖点或信息

保存到 `_briefs/项目名-brief.md`

**输出模式说明**：

| 模式 | 适用场景 | 特点 |
|------|----------|------|
| **图文笔记** | 需要靠截图、对比图、封面图承接信息 | 正文为图服务，配图清单更重要 |
| **纯文字帖** | 主要靠正文本身完成表达，图片只是辅助 | 正文必须单独成立，更强调钩子、短段、列表、互动收口 |

### Step 4：检查长文合集

路径：`~/WorkBuddy/Claw/小红书/collections/`

执行顺序：
1. 先读取已有合集列表，理解每个合集的主题边界、适用文章类型、已有封面
2. 判断当前文章是否适合并入某个已有合集

4. 如果没有合集，或没有合适合集，再新建一个合集文件并生成一张合集封面

**合集创建规则**：
- 合集名称要能覆盖一组连续内容，而不是只服务单篇文章
- 合集简介要说明收录主题、适用场景和读者收益
- 合集封面默认只在首次创建合集时生成一次
- 后续新增文章时，优先复用已有合集与封面，除非我主动要求修改

**正文中的呈现规则**：
- `# 推荐长文合集` 放在 `# 正文描述` 下方
- 至少写明合集名称、推荐理由、封面图路径

**先判断输出模式**：

适合：

1. 前三行必须承担钩子作用

6. 结尾必须有一个互动动作：

**写作原则**：
- 每段最多2-3句话，必须换行
- 口语化，像在跟朋友说话
- 有具体数字（"用了3周"比"用了一段时间"好10倍）
- 有真实细节（具体的场景、感受、操作步骤）

**表情使用原则**：
- emoji 不是硬性要求，也不是固定配额
- 是否使用、使用多少，取决于内容语气、信息密度和读者情绪
- 有情绪起伏、提醒感、步骤感、清单感的内容，可以适度使用 emoji 增强扫读体验
- 偏理性分析、专业说明、克制表达的内容，可以少用，甚至完全不用
- 如果使用，优先放在标题、小标题、列表引导、提醒句、结尾互动句，而不是塞进每一段正文
- 任何 emoji 都应服务表达，而不是为了制造“小红书感”机械堆砌

**编辑器友好语义（建议保留）**：

为了后续进入可视化编辑器后更容易匹配主题和结构，正文中优先使用这些稳定信号：

- `📌 先说结论`
- `✅ 适合谁`
- `❗ 注意`
- `👉 一个提醒`
- `1️⃣ 2️⃣ 3️⃣`
- `Q:` / `A:`
- `---`

这些不是为了制造“模板痕迹”，而是为了增强扫读效率，并给后续排版主题提供明确抓手。

### Step 6：三遍审校

**第一遍**：内容真实性（数据、产品信息）

**第二遍**：去广告感/去AI感
- 检查有没有"优质"、"卓越"、"显著提升"这类词
- 检查有没有过于完美的产品描述（真实体验有优有劣）
- 确保有真实的使用细节，不是泛泛的推荐

**第三遍**：格式和节奏
- 前三行是否足够吸引
- 分段是否够短
- 长段是否已经主动拆开
- 列表是否比普通长段更清楚
- 结尾是否有互动引导（"你们有遇到过这个问题吗"之类）

**默认组合**：3个大流量 + 4个中流量 + 3个小流量 = 10个标签

**生成原则**：
- 大流量标签负责告诉平台“这篇属于哪个大类”，不要全写泛词，也不要和正文主题无关
- 中流量标签负责补充内容主题、使用方式和目标人群，尽量贴近正文真实内容
- 小流量标签必须落到这篇文章的具体场景、工具、动作或结果，不能空泛
- 10 个标签里，优先保留“平台能理解的大类词 + 读者会搜索的主题词 + 这篇独有的场景词”
- 标签之间不要只是同义堆叠，例如 `#AI工具 #AI神器 #AI黑科技` 这种应避免
- 如果正文是工具体验，至少包含：工具类、任务类、场景类三种标签
- 如果正文是经验分享，至少包含：主题类、受众类、问题类三种标签
